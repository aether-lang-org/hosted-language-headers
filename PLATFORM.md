# Platform of the headers on this branch

This branch (`main`) carries headers captured from:

| Field | Value |
|-------|-------|
| OS    | Linux |
| libc  | glibc |
| Arch  | x86_64 |
| Distro snapshot | Debian 12 (bookworm) |

The captured-from package versions:

| Language | Source package | Version |
|---|---|---|
| Python    | `python3-dev`      | 3.11.2 |
| Lua       | `liblua5.4-dev`    | 5.4.4 |
| Perl      | `libperl-dev`      | 5.36 |
| Ruby      | `ruby-dev`         | 3.1.2 |
| duktape   | `duktape-dev`      | 2.7.0 |

## What's architecture-sensitive

Most of each header tree is portable C declarations and would compile
identically on any Unix-y target. The exceptions are configured
headers populated by each language's `./configure` at packaging time:

| Language | Configured file(s) | What configure decides |
|---|---|---|
| Python | `pyconfig.h` | sizeof(long), available syscalls, FFI conventions |
| Lua    | `luaconf.h` | LUAI_BITSINT, number type (int vs double) |
| Perl   | `config.h` + a handful in CORE/ generated by Configure | the whole world; perl assumes pervasive configure-time substitution |
| Ruby   | `ruby/config.h` (in the `ruby-arch/` subtree) | sizeof(VALUE), pointer width, asm conventions |
| duktape| `duk_config.h` | endianness, alignment, optional features |

## When to branch

For another linux-x86_64-glibc snapshot, this branch suffices —
overwrite contents, commit, retag.

For a different target (linux-arm64, linux-musl, macOS, …):
**make a new branch**, e.g. `linux-arm64-glibc`, populate from
a host running that target. The diff between branches will be
small (mostly the configured `*config.h` files above) and is
itself good documentation of the per-arch surface.

Don't try to merge multiple targets into one branch. Pick one,
own it, document it here, branch for others.
